🎯 APP CONCEPT

This is a two-sided marketplace:

1️⃣ Customer App
2️⃣ Mechanic App

Both use the same backend but different roles.

👤 CUSTOMER FLOW

User signs up / logs in using JWT.

User enables “Location while using app”.

Home screen:

Center animation: “Finding your mechanic…”

Search bar at top for manual mechanic search.

User taps “Request Mechanic”.

Backend:

Finds available mechanics within 5km using Haversine formula.

Emits real-time request via Socket.io to nearby mechanics.

First mechanic who accepts:

Becomes assigned to the request.

Customer receives:

Mechanic name

Phone number

Rating

Live location

Google Map opens:

Shows mechanic moving toward user.

After service:

Rating system

Service history saved

🔧 MECHANIC FLOW

Mechanic registers with:

Name

Shop name

Phone

Location (GeoJSON)

Working hours

Mechanic can toggle:

Available

Busy

Offline

When nearby request comes:

Popup with Accept / Reject

If accepted:

Live navigation to customer

Update live location every few seconds

📡 REAL-TIME SYSTEM

Use Socket.io.

Flow:

Customer emits:
"request_mechanic"

Server:

Finds mechanics within 5km

Emits "new_request" to those mechanics

Mechanic emits:
"accept_request"

Server:

Assigns mechanic

Emits "request_accepted" to customer

Locks request

Implement:

Request timeout (30 seconds)

Prevent double assignment

Room-based socket handling
